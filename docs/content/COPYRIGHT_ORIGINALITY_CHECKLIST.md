# Nightglass Core Set Copyright and Originality Checklist

## Purpose
Release gate to ensure base-set content remains original and avoids overlap with proprietary Fantasy Flight content.

## Blocking Criteria
- Any direct reuse of protected names, places, factions, entities, scenario titles, or proprietary terms.
- Any text snippets that are near-copy or structurally derivative from known published card or scenario text.
- Any one-to-one map, progression, or campaign cloning from protected products.

## Review Steps
1. Name screening
- Scan all content IDs and display names against prohibited-term blocklist.
- Reject records with exact or near-match protected terms.

2. Phrase uniqueness review
- Review all summaries and long-form strings for derivative language patterns.
- Rewrite text that follows known proprietary cadence or template structures.

3. Mechanic wording review
- Ensure action labels, status labels, and phase names are original to this project.
- Replace any borrowed jargon with project-native wording.

4. Thematic and structural review
- Confirm scenario premise, objective chain, and escalation logic are independently authored.
- Confirm no direct analog mapping to protected faction/story structures.

5. Signoff
- Two-person reviewer approval required (design + editorial/legal).
- Record date, reviewers, and final decision in release notes.

## Red-Flag Patterns
- Trademarked proper nouns or recognizable proper-noun variants.
- Reused encounter prose structure from published card text.
- Distinctive icon labels copied from protected UI systems.
- Character backstories that map directly to known franchise identities.
